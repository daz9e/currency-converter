# 💱 Telegram Currency Converter Bot

Telegram бот для конвертации валют между RSD (сербские динары), UAH (гривны), RUB (рубли) и EUR (евро).

## 🚀 Возможности

- Выбор валюты по умолчанию при первом запуске
- Конвертация просто по числу (использует валюту по умолчанию)
- Конвертация с указанием валюты (например, `100 EUR`)
- Автоматическое обновление курсов валют через API
- Сохранение настроек пользователей в SQLite
- Полностью в Docker контейнере

## 📋 Требования

- Docker
- Docker Compose
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather))

## 🛠️ Установка и запуск

### 1. Клонируйте репозиторий или скопируйте файлы

```bash
cd /path/to/money-convert
```

### 2. Создайте .env файл

```bash
cp .env.example .env
```

Отредактируйте `.env` и укажите ваш токен бота:

```env
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
```

### 3. Запустите бота

```bash
docker-compose up -d
```

### 4. Проверьте логи

```bash
docker-compose logs -f
```

## 📱 Использование

1. Найдите своего бота в Telegram
2. Отправьте `/start` - выберите валюту по умолчанию
3. Отправьте число для конвертации, например: `100`
4. Или отправьте число с кодом валюты: `50 EUR`
5. Для смены валюты по умолчанию используйте `/currency`

### Примеры

```
100          # Конвертирует 100 единиц вашей валюты по умолчанию
50 EUR       # Конвертирует 50 евро
1000 RUB     # Конвертирует 1000 рублей
25.5 UAH     # Конвертирует 25.5 гривен
```

## 💾 База данных

База данных SQLite сохраняется в директории `./data/bot.db` и автоматически создается при первом запуске. Все настройки пользователей сохраняются и не теряются при перезапуске контейнера.

## 🌍 Поддерживаемые валюты

- 🇷🇸 **RSD** - Сербские динары
- 🇺🇦 **UAH** - Гривны
- 🇷🇺 **RUB** - Рубли
- 🇪🇺 **EUR** - Евро

## 📊 Источник курсов

Бот использует [ExchangeRate-API](https://exchangerate-api.com/) для получения актуальных курсов валют. Если API недоступен, используются резервные курсы.

## 🐛 Troubleshooting

### Бот не отвечает
- Проверьте, что контейнер запущен: `docker ps`
- Проверьте логи: `docker-compose logs -f`
- Убедитесь, что токен бота правильный

### База данных не сохраняется
- Проверьте права на директорию `./data`
- Убедитесь, что volume корректно примонтирован

### Курсы валют не обновляются
- Проверьте подключение к интернету
- Бот использует резервные курсы, если API недоступен
